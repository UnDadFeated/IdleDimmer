## Build

- **MSVC**: `build.bat` (auto-detects VS 2022, runs `rc.exe` + `cl.exe`)
- **LLVM-MinGW** (always rebuild `resources.o` and `setup_res.o`):
  ```cmd
  llvm-windres resources\resources.rc -O coff -o resources\resources.o
  clang++ -O2 -std=c++17 -mwindows -Os -s -mguard=cf -fms-extensions -static -o IdleDimmer.exe src\main.cpp src\MainWindow.cpp src\MainWindowDraw.cpp src\MainWindowInput.cpp src\DimmerManager.cpp src\ConfigManager.cpp resources\resources.o -lgdi32 -ld2d1 -ldwrite -ldwmapi -lole32 -luuid -lwinhttp -lversion -Wl,--dynamicbase -Wl,--nxcompat -Wl,--high-entropy-va -Wl,--subsystem,windows
  ```
- **Setup** (requires `IdleDimmer.exe` in project root ‚Äî rebuild resources *after* app exe to embed the correct version):
   ```cmd
   llvm-windres resources\setup.rc -O coff -o resources\setup_res.o
   clang++ -O2 -std=c++17 -mwindows -Os -s -mguard=cf -static -o IdleDimmer-Setup-v1.6.3.exe src\setup.cpp resources\setup_res.o -lole32 -lshell32 -ladvapi32 -luuid -lcomctl32 -lversion -Wl,--dynamicbase -Wl,--nxcompat -Wl,--high-entropy-va -Wl,--subsystem,windows
   ```

- Output: `IdleDimmer.exe` (~150 KB), `IdleDimmer-Setup-v1.6.3.exe` (~280 KB with embedded exe), `dist\IdleDimmer_1.6.3.0_x64.msix` (~880 KB, for Microsoft Store)

## Architecture

- **Singletons**: `MainWindow::Instance()` (settings panel), `DimmerManager::Instance()` (overlays)
- **4 source files** in `src/`:
  - `main.cpp` ‚Äî entry, COM init, Per-Monitor DPI v2, single-instance mutex (`Global\IdleDimmerMutex`), message loop
  - `MainWindow.*` ‚Äî D2D-rendered settings panel with slider cards and toggle switches
  - `DimmerManager.*` ‚Äî per-monitor layered overlay windows, fade animation (16ms timer)
  - `ConfigManager.*` ‚Äî config I/O (simple JSON scanner, **not** a full JSON parser)
- **Window classes**: `IdleDimmerMainClass` / `IdleDimmerOverlayClass` / `IdleDimmerSetupClass`
- **Overlay style**: `WS_EX_LAYERED | WS_EX_TRANSPARENT | WS_EX_NOACTIVATE | WS_EX_TOOLWINDOW | WS_EX_TOPMOST`
- **Config file**: `%APPDATA%\IdleDimmer\dimmer.ini` (JSON-like, hand-parsed ‚Äî no external parser library)

## Value Ranges

- Monitor dim: `0`‚Äì`90` (slider `value / 90.0f` ‚Üí float `0.0`‚Äì`1.0`)
- Idle dim level: `0`‚Äì`100` (slider `value / 100.0f`)
- Idle timeout: `1`‚Äì`60` minutes (slider `(value - 1) / 59.0f`)

## Layout Rules (preserve these)

- **Sequential Y**: `yOffset = last_element.rect.bottom + margin` ‚Äî no hardcoded offsets like `+= 90`
- **Label X**: Monitor slider labels at `slider.rect.left + 60.0f` (X=80) to avoid 34√ó18 toggle switches cutting through text. Idle sliders (no toggle) use `+20.0f`.
- **Checkbox text clipping**: Column 1 clipped at `m_windowWidth / 2.0f - 10.0f`, Column 2 at `m_windowWidth - 20.0f`
- **Window height**: calculated dynamically in `UpdateLayout()` ‚Äî `m_windowHeight = yOffset + 55`
- **DISPLAY section**: Only WarmTint (col 0) and LightMode (col 1) checkboxes remain ‚Äî FocusMode and ShowBoundaries removed.

## Key Behaviors

- **Startup**: `dimmingEnabled` defaults to `false` ‚Äî never auto-dim on launch
- **Per-monitor**: Each monitor has its own toggle and slider ‚Äî no group mode
- **Auto-enable**: Dragging/arrowing/scrolling a monitor slider auto-enables `dimmingEnabled` if it was off
- **Undo**: Session-start config snapshot (`m_backupConfig`); "Undo Changes" button in top-right header restores it
- **Device loss**: `D2DERR_RECREATED` ‚Üí discard + recreate graphics resources (`MainWindow.cpp:626-629`)
- **Hot-plug**: `WM_DISPLAYCHANGE` triggers `RefreshMonitors()` ‚Üí rebuild overlays, preserve saved per-monitor values
- **Cursor**: Only hides during idle dimming (`m_isIdleState`), never during active dimming (`DimmerManager.cpp:186`)
- **Version**: Read from PE version resource at runtime via `GetOwnVersion()` helper ‚Äî no hardcoded version strings in code

## Video Detection Rules (`CheckVideoPlayback`)

- **State only changes on full-check ticks** (`m_videoCheckTick % 5 == 0`, every 5 seconds)
- **Per-Monitor Focus**: Dimming bypass is now monitor-specific.
- **Audio & Video Association**: When a media player process (from `m_blockedApps`) or a major web browser (`chrome.exe`, `firefox.exe`, `msedge.exe`, `opera.exe`, `brave.exe`) is actively playing audio, we query its visible, non-minimized (`!IsIconic`) windows to locate the monitor(s) they reside on. Only those specific monitors are marked with `hasVideo = true` to bypass dimming.
- **No Background Audio Bypass**: If the media or browser window is minimized, or if only a pure background audio app (like Spotify, which is not a browser or media player) is playing audio, the monitors will dim normally since the user is not watching the monitor.
- On lightweight ticks: checks if the foreground window is fullscreen, and if so, prevents only that specific monitor from dimming.
- `IsForegroundWindowFullscreen()` excludes `IdleDimmerMainClass` and `IdleDimmerOverlayClass`

## Overlay Priority

- Idle > video > dimming. If idle, overlays dim even if blocked app is detected. Active dimming engages only when neither idle nor video is active.

## Defender Heuristics (do not reintroduce these patterns)

- `HWND_BROADCAST` + `SC_MONITORPOWER`: use `SendMessageW(GetDesktopWindow(), ...)` or target `Shell_TrayWnd` instead
- `OpenProcess` / `QueryFullProcessImageNameW`: only inside `doFullCheck` (every 5th tick), not every second
- `LoadLibraryW("shell32.dll")` / `FreeLibrary`: only inside `doFullCheck` (every 5th tick)
- `EnumChildWindows` + process name resolution in timer loop: throttled to 5-second cadence
- No `SetSystemCursor`, no `CreateToolhelp32Snapshot` ‚Äî use `ShowCursor` and `IAudioSessionEnumerator`
- Code signing not implemented yet; submit to MSRT false positive portal if needed

## Undo (one-by-one)

- Each change pushes to `m_undoStack`. Each undo pops the latest entry and applies it.
- NOT a single snapshot restore ‚Äî `m_backupConfig` is the session-start snapshot for the first use case, but subsequent changes stack individually.

## Gotchas

- `std::wifstream` with `.c_str()` not `std::wstring` ‚Äî MinGW may fail otherwise (`ConfigManager.cpp:38,121`)
- Resource manifest: `1 24 "manifest.xml"` ‚Äî standard type 24 for MSVC + MinGW compat
- `m_pRenderTarget->SetColor()` is called each `OnPaint()` to switch theme ‚Äî brushes are created once, colors swapped at render time
- Version resources: update all of `resources.rc`, `setup.rc`, and `manifest.xml` on each version bump. Also update the hardcoded `VER` string in `setup.cpp`.
- Version: define `APP_VERSION` constant in `MainWindow.cpp` ‚Äî keep in sync with resources.rc, setup.rc, manifest.xml, and setup.cpp `VER` on every bump.
- MinGW pragma `comment(lib, ...)` is silently ignored ‚Äî use `-l` flags on the clang command line instead.

## Releases & Headless Publishing

To publish a release to the GitHub web interface when local authentication for the GitHub CLI (`gh`) is unavailable (e.g. inside a headless sandbox or build pipeline), you can leverage the local Git Credential Manager token without storing it in files:

1. **Retrieve the token**: Query the Git Credential helper to get your active GitHub OAuth/PAT token.
   - Using Python:
     ```python
     import subprocess
     p = subprocess.Popen(['git', 'credential', 'fill'], stdin=subprocess.PIPE, stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True)
     stdout, _ = p.communicate(input="protocol=https\nhost=github.com\n\n")
     for line in stdout.splitlines():
         if line.startswith("password="):
             token = line.split("=", 1)[1]
             print(token)
     ```
   - Using PowerShell:
     ```powershell
     $output = @("protocol=https", "host=github.com", "") | git credential fill
     $tokenLine = $output | Where-Object { $_ -like "password=*" }
     $token = $tokenLine.Substring(9)
     ```

2. **Execute Headless Release**: Set the token to the `GH_TOKEN` environment variable so `gh` uses it directly, bypassing scope verification login limits:
   ```cmd
    $env:GH_TOKEN="<retrieved_token>"
    gh release create v1.6.3 IdleDimmer-Setup-v1.6.3.exe --title "v1.6.3" --notes "Release notes go here"
   ```

3. **Delete old tags before re-releasing**: `git tag -d vX.Y.Z; git push origin --delete vX.Y.Z` when replacing a release.

4. **Release Checklist**: Always update the git repo (add, commit, push), push the new setup `.exe` to GitHub releases, update references in `README.md`, and delete old installer files and outdated git tags/releases.
